SETUP OPTIONS:
--------------

Option 1: Service Account (REQUIRED for Automated Benchmark Runs)
-------------------------------------------------------------------
⚠️ IMPORTANT: For automated benchmark runs, service accounts are REQUIRED because 
   agents cannot interact with browser authentication prompts.

Setup Steps:
1. Create a service account in Google Cloud Console (using agenthle.sv@gmail.com)
2. Enable Google Earth Engine API for the service account
3. Grant Earth Engine access to the service account email in Earth Engine Console
4. Download the service account JSON key file
5. Set environment variables before running benchmark:
   ```bash
   export GEE_CREDENTIALS_PATH="/path/to/service-account-key.json"
   export GEE_SERVICE_ACCOUNT="service-account@project.iam.gserviceaccount.com"
   ```

In Python code:
```python
import ee
import os
credentials = ee.ServiceAccountCredentials(
    os.environ.get("GEE_SERVICE_ACCOUNT"),
    os.environ.get("GEE_CREDENTIALS_PATH")
)
ee.Initialize(credentials)
```

✅ No browser interaction needed - fully automated!

Option 2: Regular Account (For Local Manual Testing Only)
----------------------------------------------------------
⚠️ WARNING: This method requires interactive browser authentication (clicking, confirming).
   Agents CANNOT do this automatically. Only use for manual local testing.

```python
import ee
ee.Authenticate()  # Opens browser - you must click and confirm
ee.Initialize()
```
